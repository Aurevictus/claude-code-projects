# Claude Code — Project Instructions

## Git & GitHub Workflow

After completing any meaningful unit of work, always commit and push to GitHub:

```bash
git add <changed files>
git commit -m "concise description of what changed and why"
git push
```

**Rules:**
- Commit after every feature, fix, or notable change — never let work pile up uncommitted
- Write clear, specific commit messages (what changed + why, not just "update file")
- Always push after committing so GitHub stays in sync and work is never lost
- Stage specific files by name rather than `git add -A` to avoid accidentally committing unintended files
- Never force-push or amend commits that have already been pushed
